Deterministic, Time-Synchronized Transport of Professional Audio, Video, and Data over Ethernet

PMoIP represents the convergence of three critical standards that together enable real-time, professional-grade media transport over standard IP networks. This unified framework ensures deterministic delivery, precise synchronization, and seamless interoperability for broadcast, live production, and professional media applications.

By combining SMPTE ST 2110 (media essence transport), AES67 (audio interoperability), and IEEE TSN (network-level determinism), PMoIP creates a comprehensive solution for modern media workflows.

  • ST 2110-20: Uncompressed Video
  • ST 2110-30: PCM Audio (based on AES67)
  • ST 2110-40: Ancillary Data

üîç Deep Dive: Components Explained

SMPTE ST 2110 Details +

Professional Media Over Managed IP Networks

ST 2110-10: System Timing & Synchronization

  • Uses Precision Time Protocol (PTP) per SMPTE ST 2059-2
  • Ensures all streams are precisely aligned
  • Microsecond-level accuracy across all network devices

ST 2110-20: Uncompressed Video

  • Transports raw video as RTP streams
  • Supports multiple color spaces (BT.709, BT.2020, etc.)
  • Flexible sampling structures (4:2:2, 4:4:4)
  • Bit depths from 8 to 12 bits
  • Supports SD, HD, UHD, and 4K formats

ST 2110-30: PCM Audio

  • Built on AES67 specification
  • Subset of AES67 with stricter constraints
  • Requires SMPTE ST 2059-2 PTP profile
  • Zero offset between media and RTP clock
  • Supports up to 64 channels per stream

ST 2110-40: Ancillary Data

  • Carries closed captions, subtitles, timecode
  • Metadata and control information
  • Synchronized with video and audio streams
AES67 Audio Interoperability +

High-Performance Streaming Audio-over-IP

Core Principles:

  • Layer 3 (IP) audio transport using RTP
  • Uncompressed linear PCM audio
  • Sample rates: 44.1, 48, 88.2, 96 kHz
  • Bit depths: 16 or 24 bits
  • Low latency: typically <5ms

Synchronization:

  • IEEE 1588-2008 PTP for clock synchronization
  • Ensures sample-accurate playback across devices
  • Network-wide media clock distribution

Session Management:

  • SDP (Session Description Protocol) for stream description
  • SAP (Session Announcement Protocol) for discovery
  • Multicast (IGMPv3) and unicast support
IEEE TSN Network Determinism +

Time-Sensitive Networking for Professional Media

IEEE 802.1AS (gPTP):

  • Generalized Precision Time Protocol
  • Microsecond-level synchronization across network
  • Evolution of IEEE 1588 for Ethernet
  • Eliminates timing drift and jitter

IEEE 802.1Qbv (Time-Aware Shaper):

  • Schedule-based traffic shaping
  • Guarantees bandwidth for time-critical streams
  • Prevents network congestion
  • Enables deterministic latency

IEEE 802.1Qbu (Frame Preemption):

  • Allows high-priority frames to interrupt lower-priority traffic
  • Reduces worst-case latency
  • Critical for real-time media streams

IEEE 802.1CB (Redundancy):

  • Frame replication and elimination
  • Seamless redundancy for mission-critical applications
  • Zero packet loss during failures
Integration: How They Work Together +

Unified PMoIP Framework

Synchronization Layer:

  • IEEE 802.1AS (TSN) provides network-wide time synchronization
  • SMPTE ST 2059-2 adapts PTP for broadcast applications
  • All streams (video, audio, data) locked to common clock
  • Sub-microsecond accuracy ensures lip-sync and phase coherence

Transport Layer:

  • SMPTE ST 2110 separates essence into independent streams
  • AES67 ensures audio interoperability across vendors
  • RTP/UDP provides real-time transport with minimal overhead
  • Each stream can be routed independently

Quality of Service (QoS):

  • TSN traffic shaping (802.1Qbv) reserves bandwidth
  • Frame preemption (802.1Qbu) prioritizes media streams
  • Guaranteed latency and jitter performance
  • Eliminates packet loss for critical streams

Resilience & Redundancy:

  • TSN frame replication (802.1CB) provides seamless failover
  • Dual-network architectures for high availability
  • Hot-standby configurations
  • Zero downtime during network failures
